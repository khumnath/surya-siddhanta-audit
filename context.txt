# Surya Siddhanta Audit: Project Context

This project is a high-fidelity realization of the pure astronomical formulas of the ancient Surya Siddhanta (SS), implemented in TypeScript. It serves as both a functional astronomical engine and an academic platform for auditing traditional calendrical logic against modern JPL-based data.

## Core Philosophical Pillar
**Zero-Lookup Architecture**: Every astronomical value (planetary position, tithi transition, ingress, etc.) is derived formulaically from prime parameters. The library contains no historical lookup tables.

## System Components

### 1. Traditional Engine (`src/lib/surya-siddhanta/`)
The primary mathematical core implementing the Siddhantic formulas.
- **Precision**: Calculations maintain a $10^{-6}$ degree fidelity against authoritative texts.
- **Verification**: Documented with JSDoc headers that inject authoritative Sanskrit proofs during the build process.

### 2. Modern Audit Bridge (`src/lib/modern/`)
A bridge to the NASA-JPL based `astronomy-engine`, used for calculating comparative drift and calibrating ayanamsha models.

### 3. Diagnostic Suite (`scratch/`)
A purpose-built verification pipeline designed to prevent mathematical regressions.
- **`dump_data.ts`**: Snapshots canonical astronomical states for calibration.
- **`verify_data.ts`**: An interactive utility for 1:1 regression audits against established baselines.

### 4. Authoritative References (`reference/`)
Consolidated primary sources used to validate constants and formulas, including the Sanskrit original and the authoritative Burgess translation/commentary.

## Current State
The engine is currently in a "Stably Calibrated" state, meaning it handles cross-era calculations (from Kali Yuga epoch to the present day) with internally consistent mathematical precision.
