# Surya Siddhanta Audit: Definitive Project Context

This document is the authoritative technical reference for the Surya Siddhanta (SS) Audit project. It is designed to provide future AI agents and developers with an immediate, deep understanding of the system's architecture, philosophy, and current status.

## 1. Project Philosophy & Core Principles
- **Zero-Lookup derivation**: The engine rejects hard-coded astronomical tables. Every tithi, longitude, and ingress is computed from prime motion constants defined in `verses.json`.
- **Mathematical Parity**: The system is calibrated to match modern JPL-based data (`astronomy-engine`) with a $10^{-6}$ degree tolerance, using a configurable Ayanamsha bridge.
- **Academically Grounded**: Every significant computational function is linked to a canonical Sanskrit proof from the Burgess/Surya Siddhanta text.

## 2. Technical Stack
- **Core**: TypeScript (Targeting ESNext).
- **Frontend**: React + Vite (Custom Premium Theme).
- **Time/Date**: Luxon (Native ISO and UTC handling).
- **Audit**: `astronomy-engine` (Modern JPL Bridge).
- **Documentation**: TypeDoc with custom scriptural injection scripts.

## 3. Directory Architecture
- `/src/lib/surya-siddhanta/`: The traditional engine.
    - `center/`: Core celestial logic (mean motions, anomalies, true longitudes).
    - `calendar/`: Panchanga elements (Tithi, Nakshatra, Karana, Yoga).
    - `time/`: Ahargana (Siddhantic Julian Day) and epoch conversion logic.
    - `geography/`: Terrestrial longitudes, latitudes, and timezones.
- `/src/lib/modern/`: The modern audit bridge.
- `/scratch/`: The diagnostic suite. 
    - `dump_data.ts`: Snapshot generation.
    - `verify_data.ts`: Regression auditing.
    - `internal/`: Archived legacy tests.
- `/reference/`: Authoritative primary source texts (PDF/TXT).
- `/scripts/`: Automated build and documentation logic.

## 4. Proprietary Diagnostic Pipeline
To prevent mathematical drift during refactoring:
1.  **Generation**: `npm run dump` (via `scratch/dump_data.ts`) creates a JSON snapshot of the engine's current state.
2.  **Regression**: `npm run verify` (via `scratch/verify_data.ts`) performs a deep-comparison between a baseline snapshot and the live code, reporting any delta in Ahargana or planetary longitudes.

## 5. Documentation Build Process
The documentation is unique because it is dynamically enriched:
- **`resolve-siddhanta-logic.js`**: Scans the source files for specifically formatted JSDoc tags and injects Sanskrit verses from `verses.json`.
- **Branding**: The build includes custom CSS and JS injection for a premium UI (theme toggle, branded footers).
